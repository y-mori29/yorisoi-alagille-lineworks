# 親から子へのアカウント引き継ぎ設計メモ

作成日: 2026-06-21

## 背景

2026-06-01 10:00のミーティングで、アラジール症候群版では「最初は親が登録・管理し、将来的に子ども本人へ内容を引き継ぐ」話が出た。

MVPでは完全実装しないが、あとから壊さず実装できるように、患者データに保護者と本人の概念を分けて持たせる。

## MVPで入れるデータ構造

```json
{
  "diseaseId": "alagille",
  "childProfile": {
    "name": "",
    "birthDate": "",
    "sex": ""
  },
  "guardian": {
    "displayName": "",
    "relationship": "",
    "lineWorksUserId": ""
  },
  "futureHandoff": {
    "status": "not_started",
    "childUserId": "",
    "requestedAt": null,
    "completedAt": null
  }
}
```

## 画面上の扱い

- MVPでは「将来、本人へ引き継げるように準備中」程度の表現に留める。
- 記録の主語は「お子さんの記録」、操作主体は「保護者」に分ける。
- 成人後・本人利用開始時の権限変更は、本実装時に改めて設計する。

## 後で決めること

- 本人確認の方法
- 保護者の同意と本人の同意
- 保護者が見続けられる範囲
- 子ども本人が成人した後の権限
- 患者会/運営者がサポートできる範囲
