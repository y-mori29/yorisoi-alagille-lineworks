## 3. 実装順

1. 初期登録、健康記録対象者の選択、年齢表示、固定名残存を修正する
2. 薬・成長をCloud Runへ反映し、検査値と同じ分離型チュートリアルを追加する
3. 診察録音を現在のアカウント・家族・対象者モデルへ実装し直す
4. 日々の様子、次回受診、次に聞くことを患者別CRUDへする
5. 最近の変化と先生に見せる画面を各APIの集約画面へする
6. 写真横断表示、Q&A、機能別チュートリアルを本導線へ接続する
7. 招待一覧・取消・状態表示を追加し、家族共有の運用を完成させる
8. Chromeで登録から共有・記録・編集削除までの実画面回帰を行う

## 4. Cloud Run前の技術条件

- `backend/Dockerfile` からイメージをビルドできる
- `deploy_env.yaml` はリポジトリへコミットせず、FirestoreデータベースID、GCSバケット、必要なAPI設定を安全に渡す
- 現在のCORS全許可と未認証公開を、本番相当の検証範囲に合わせて見直す
- 本番保存へ切り替える前に、`DEMO_MODE=0` でFirestoreの家族・患者・記録が分離されることを確認する
- 検査画像はbase64をFirestoreへ直接持たせず、GCSへ保存して参照を記録する
- OCR未設定時は「読み取れないため手入力または写真のみ保存」と利用者に伝える
- 削除対象の画像も、記録削除時にGCS側の扱いを決める
- 医療情報を扱うため、アクセスログ・エラーログへ検査値や画像本文を出さない
- OCRはVertex AIを使わず、Gemini Developer APIを `@google/genai` から直接呼ぶ
- Gemini APIキーはSecret Manager `DEV_GEMINI_API_KEY` からCloud Runへ参照し、値をソースやデプロイ引数へ書かない
