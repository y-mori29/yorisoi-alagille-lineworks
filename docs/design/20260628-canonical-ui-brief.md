## この文書の目的

画像生成・UI実装・説明資料作成の前提が散らからないように、2026-06-28時点で合意できたUI方針を短く固定する。

この文書は詳細仕様書の置き換えではない。詳細は `docs/specs/20260624-expansion/` を正本とし、この文書は「どのUI方向で作るか」を示す入口として使う。

## 合意済みの正本方針

ホームUIは、D案を骨格にした統合案を第一候補にする。

- D案: 「今日のこと」「検査・お薬」「診察に持っていく」の3エリア構成
- B案: 家族共有と患者切り替えの分かりやすさ
- E案: 最近の変化をホーム下部で見返せるサマリー
- C案: 「先生に見せる」は独立した強い画面として扱う
- F案: 通常ホームではなく、初回ガイド・LINE WORKS会員向け説明として扱う

## プロダクトの一言

アラジール症候群のお子さんとご家族が、診察・検査・薬・成長・日々の様子を家族で残し、必要な記録だけを診察時に見せられる、LINE WORKSから開く家族ノート。

## 体験の中心

「全部を管理する」ではなく、「必要な時に残して、家族で見返して、先生に伝えやすくする」。

特に大切にすること:

- 医療判断ではなく、診察で伝えるための記録補助である
- 患者本人と家族が受け取る体験を中心にする
- 父母など複数家族で同じ記録を見られる
- 複数のお子さん・患者さんがいる場合にも切り替えられる
- 入力を強制せず、写真・録音・短いメモから始められる

## ホームの構造

1. ヘッダー
   - ブランド名
   - 現在の患者
   - 患者切り替え
   - メニュー

2. 家族共有の安心表示
   - 「お母さん・お父さんと共有」
   - 共有状態を小さく見せる

3. 今日の入口
   - 見出し: 「今日は何をしますか」
   - 大分類1: 今日のことを残す
     - 録音で診察メモ
     - 日々の様子
   - 大分類2: 検査・お薬を確認
     - 検査値
     - お薬メモ
   - 大分類3: 診察に持っていく
     - 先生に見せる
     - 次に聞くこと

4. 最近の変化
   - かゆみ、検査値、お薬などの短いサマリー
   - 医学的な判定ではなく、前回からの記録メモとして表示

5. 成長・予定
   - 身長、体重
   - 次回受診

6. 下部ナビ
   - ホーム
   - カレンダー
   - 写真
   - 設定

## 画面ごとの役割

| 画面 | 主目的 | 主要アクション | 注意 |
| --- | --- | --- | --- |
| 初回ガイド | LINE WORKSから使い始める流れを伝える | よりそいを開く、患者を選ぶ | F案を発展させる |
| 患者切り替え | 対象患者を選ぶ | 切り替え、追加 | 記録混在を防ぐ |
| ホーム | 今日使う入口を選ぶ | 録音、検査、お薬、日々、先生に見せる | 統合案を正本候補にする |
| 診察メモ | 診察内容を残す | 録音、AI整理、次に聞くこと | AI補完しすぎない |
| 検査値 | 検査結果を写真から確認する | 撮影、OCR確認、修正、保存 | 判定色を強く出さない |
| お薬メモ | 薬を写真・名前・履歴で残す | 撮影、登録、継続/中止 | 薬を勝手に提案しない |
| 成長 | 身長体重を見返す | 入力、推移確認 | 診断ではなく記録 |
| 日々の様子 | 写真・動画・短いメモで残す | 写真、動画、カテゴリ、メモ | リアルすぎる症状表現を避ける |
| 先生に見せる | 診察時に必要な記録だけまとめる | 範囲選択、コピー、印刷 | スマホ提示でも読める |
| 家族共有 | 父母などが同じ記録を見る | 招待、権限確認 | 初期は父母中心 |
| Q&A | 保存・費用・安全性の不安を解消 | 質問を見る | 専門用語を避ける |

## 言葉のルール

使う言葉:

- 残す
- 見返す
- 家族で共有
- 先生に伝えやすく
- 診察につなぐ
- 記録の補助
- 必要な時だけ

避ける言葉:

- 診断します
- 判定します
- 正常/異常を断定
- 患者管理
- 管理対象
- 強制的な記録

## 画像生成のルール

- 画面全体画像は、実装の見た目と説明用に使う
- 実装では、文字・カード・ボタン・ナビをHTML/CSSで再構築する
- 実装素材は、文字なし・単体・切り出しやすい水彩アセットを優先する
- 医療判断に見える赤色警告や重い病院表現は避ける
- LINE WORKSの詳細仕様が未確定なものは「予定」「候補」「次段階」として表現する
